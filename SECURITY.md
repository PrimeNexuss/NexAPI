# Security Policy

## ⚠️ IMPORTANT DISCLAIMER

**NexAPI is a deliberately vulnerable application designed for educational and authorized security research purposes only.**

### Intended Use

- **Educational Learning**: Understanding OWASP API Security Top 10 vulnerabilities
- **Security Training**: Practicing API penetration testing methodologies
- **Authorized Research**: Conducting security assessments in controlled environments
- **Academic Research**: Studying vulnerability patterns and exploitation techniques

### Prohibited Use

- **Production Deployment**: NEVER deploy this application on public servers or production environments
- **Unauthorized Testing**: Do NOT use against systems without explicit written authorization
- **Malicious Activities**: Any use for malicious purposes is strictly prohibited
- **Public Exposure**: Do NOT expose vulnerable endpoints to the internet

### Lab Environment Requirements

- **Isolated Network**: Run in an isolated virtual machine or container
- **Local Access Only**: Bind to localhost (127.0.0.1) or use VPN for remote access
- **No Real Data**: Use only dummy/test data - never real financial or personal information
- **Firewall Rules**: Restrict access to authorized testers only

### Vulnerability Disclosure

This project intentionally contains security vulnerabilities for educational purposes. These are NOT security issues to be reported:

- All documented vulnerabilities in the `exploits/` directory are intentional
- The application is designed to be vulnerable - this is a feature, not a bug
- Do not submit vulnerability reports for intentionally implemented weaknesses

### Responsible Disclosure

If you discover **unintended** security vulnerabilities (not documented in the exploit guides):

1. **Private Disclosure**: Contact the maintainer privately via GitHub Security Advisory
2. **Provide Details**: Include clear reproduction steps and impact assessment
3. **Allow Time**: Give the maintainer 90 days to address the issue before public disclosure
4. **Coordinated Disclosure**: Work with the maintainer on coordinated disclosure timeline
